# Backend 開發紀錄 - 岩壁計算機 (SA-HAHAHA)

## 📌 目前進度摘要

後端「資管系畢業初審大腦」已完成核心架構實作。具備雙系統自動爬取（歷年成績 + 當期選課）、系所判斷、精確學分與類別分類（全人、通識次領域、EMI）、以及動態擋修預警功能。

---

## ✅ 已實作功能 (Implemented)

### 1. 基礎架構

- [x] **CORS 設定**: `main.py` 已加入 `CORSMiddleware`。
- [x] **Health Check**: 新增 `/health` 接口。
- [x] **測試儀表板**: 新增 `index.html` 供快速開發測試 (http://localhost:8000/)，支援通識各領域細項進度條顯示。

### 2. 雙系統爬蟲服務 (`FjuScraperService` & `EstuScraper`)

- [x] **歷年成績 (SIS)**: 透過 JWT 登入對接輔大 SIS 內部 API (JSON) 取得歷史成績。
- [x] **當期選課 (ESTU)**: 模擬登入選課清單系統 (ASP.NET WebForms)，解析 HTML 表格取得「修課中 (enrolled)」的課程。
- [x] **自動整併去重**: 自動將兩邊系統的資料合併，以選課系統的精確分類覆蓋歷史成績的預設分類。
- [x] **SSL 豁免**: 處理學校系統憑證過期問題。

### 3. 審查大腦 (`Auditor Logic`)

- [x] **修課狀態追蹤**: 支援 `passed` (及格)、`failed` (不及格/停修)、`enrolled` (修課中/未評定成績) 三種狀態。
- [x] **精確分類 (ESTU 支援)**:
  - 擷取官方標註的「通識領域」與「次領域」(如：`通識-人文 (PT歷史與文化)`)。
  - 自動辨識並標註 `(EMI)` 全英語授課課程。
  - 依照官方選別精確區分「必修」與「選修」。
- [x] **全人教育整合**:
  - 體育類課程自動歸類至「核心課程」。
  - 外國語文類課程自動歸類至「基本能力」。
- [x] **動態擋修預警**: 結合 `enrolled` 狀態，若正在修習先修課程（如系統分析與設計），會給予溫馨提示而非嚴重擋修警告。

## 📄 學分分類架構與關鍵字對照表

| 分類         | 門檻 (學分) | 判斷來源 / 邏輯                                 |
| :----------- | :---------: | :--------------------------------------------- |
| **必修**     |     64      | 官方選別為「必」，或排除全人教育後的剩餘必修課程 |
| **選修**     |     32      | 官方選別為「選」，或任何系選修與自由選修       |
| **全人核心** |      8      | 大學入門、人生哲學、專業倫理、企業倫理 + 體育類課程 |
| **基本能力** |     12      | 國文 + 外國語文類課程                          |
| **通識**     |     12      | 官方通識領域標註 (含人文、自然、社會科學及次領域) |
| **EMI**      |      -      | 官方課程標記含 `EMI` 或名稱含 `英-專業`          |
